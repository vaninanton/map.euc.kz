# Идеи для развития map.euc.kz

Бэклог идей по итогам анализа проекта (2026-07-02). Не обязательства — кандидаты на проработку.
✅ = взято в работу.

## Админка

- ✅ **Дашборд `/admin`** — точки/маршруты (+суммарные км), райдеры за сегодня/7 дней/месяц/год, sparkline активности за 30 дней, pending-заявки, ошибки рассылок, health-check webhook'а бота. RPC `get_admin_dashboard_stats`.
- ✅ **Бейдж pending-заявок** в боковом меню админки.
- **Детектор дубликатов точек** — при модерации заявки показывать существующие точки в радиусе ~200 м (haversine уже есть в `geoMath.ts`).
- **Мини-карта в списках** точек/маршрутов (hover/раскрытие строки) — компоненты `AdminPointLocationMap`/`AdminRoutePolylineMap` уже готовы.
- **`updated_at` + простой аудит** на `map_points`/`map_routes` (у `map_events` триггер уже есть) — блок «что менялось за неделю» на дашборде.
- **Поиск осиротевших файлов Storage** — фото без записи в БД (после сбоев отката) и записи без файла.
- **Массовые операции** в списках: выделить несколько строк → toggle disabled.

## Публичный функционал

- **Heatmap «где катают»** — история `telegram_locations` копится бессрочно; RPC-агрегация в анонимную сетку (без user_id) → публичный слой карты. Приватность сохраняется, данных уже много.
- **Публичная страница новостей `/news`** — `map_news` сейчас видна только админам; открыть RLS на чтение (`deleted_at IS NULL`) — почти бесплатно.
- **GPX-экспорт маршрута** — кнопка «Скачать GPX» (координаты с высотами уже есть). Чистая функция, легко тестируется.
- **ICS-экспорт даты события** — «Добавить в календарь» на странице события.
- **Web Push о новых событиях** — Service Worker уже есть; нужна edge-функция для рассылки push-уведомлений.
- **RSVP с сайта** — кнопка «Участвую» на `/events/:id` → deep-link в бота (`t.me/<bot>?start=rsvp_<date_id>`), без веб-аутентификации (участники живут в `telegram_profiles`).
- **Личная статистика райдера** — на `/m/telegramuser/:id` показать км за неделю/месяц из его треков.

## Идеи из старой базы знаний (техдолг/перфоманс)

- Кластеризация точек на низких зумах (когда точек станет много).
- Vector tiles вместо GeoJSON-источников при 50K+ фич.
- LRU вместо FIFO для кэша тайлов в Service Worker.
- Офлайн-персист черновика заявки (IndexedDB, отправка при реконнекте).
